# 华为鸿蒙广告SDK示例工程 - 基础工程文件描述

## 一、工程概述

本项目是基于华为鸿蒙HarmonyOS的广告SDK示例工程，展示了如何集成和使用华为鲸鸿动能广告服务。工程使用ArkTS语言开发，支持多种广告类型的展示和管理。

**工程基本信息：**
- **包名**: com.huawei.ads.clientdemo
- **版本**: 1.0.0
- **支持的设备**: 手机（phone）、平板（tablet）
- **开发语言**: ArkTS (鸿蒙声明式UI开发语言)
- **目标SDK**: HarmonyOS AdsKit

### 3.3 事件处理

#### 3.3.1 event/RewardAdStatusHandler.ets - 激励广告状态处理器
**文件路径**: `entry/src/main/ets/event/RewardAdStatusHandler.ets`

**功能描述**:
处理激励视频广告的状态变化事件，通过公共事件机制订阅广告SDK的状态通知。

**核心方法**:
- `registerPPSReceiver()`: 注册状态监听器
  - 订阅事件: `com.huawei.hms.pps.action.PPS_REWARD_STATUS_CHANGED`
  - 监听状态: 打开、点击、关闭、奖励发放、视频播放等
- `unRegisterPPSReceiver()`: 注销状态监听器

**使用场景**:
当用户完整观看激励视频后，通过监听`AD_REWARDED`状态获取奖励信息。

#### 3.3.2 event/InterstitialAdStatusHandler.ets - 插屏广告状态处理器
**文件路径**: `entry/src/main/ets/event/InterstitialAdStatusHandler.ets`

**功能描述**:
处理插屏广告的状态变化事件，订阅插屏广告的打开、关闭、点击等状态。

**核心方法**:
- `registerPPSReceiver()`: 注册插屏广告状态监听
- `unRegisterPPSReceiver()`: 注销监听器

#### 3.3.3 event/TimeOutHandler.ets - 超时处理器
**文件路径**: `entry/src/main/ets/event/TimeOutHandler.ets`

**功能描述**:
处理广告加载超时的逻辑，主要用于开屏广告场景，避免长时间等待。

### 3.4 视图模型

#### 3.4.1 viewmodel/AdsViewModel.ets - 广告视图模型
**文件路径**: `entry/src/main/ets/viewmodel/AdsViewModel.ets`

**功能描述**:
封装广告加载和展示的核心业务逻辑，是整个广告示例工程的核心控制器。

**核心属性**:
```typescript
@ObservedV2
export class AdsViewModel {
  @Trace ads: advertising.Advertisement[] = [];        // 广告数据数组
  adOptions: advertising.AdOptions = {};               // 广告请求选项
  adDisplayOptions: advertising.AdDisplayOptions = {   // 广告展示选项
    mute: true,          // 是否静音
    refreshTime: 30000   // 轮播刷新时间（毫秒）
  };
  navPathStack: NavPathStack;  // 导航栈
  private context: common.UIAbilityContext;  // 应用上下文
}
```

**核心方法**:

1. **loadAd(adRequestParams: advertising.AdRequestParams)**
   - 功能: 加载单个广告
   - 参数: 广告请求参数（包含广告ID、类型、尺寸等）
   - 流程:
     - 创建AdLoader对象
     - 设置加载监听器（成功/失败回调）
     - 调用loadAd方法请求广告
   - 特殊处理:
     - 激励广告(REWARD): 加载成功后直接调用showAd展示
     - 插屏广告(INTERSTITIAL): 加载成功后直接展示
     - 其他广告: 加载成功后存入ads数组供后续使用

2. **loadAdWithMultiSlots(adRequestParamsArray)**
   - 功能: 批量加载多个广告位的广告
   - 使用场景: 需要同时展示多个不同广告位的场景

3. **routeToHome()**
   - 功能: 路由到主页
   - 使用场景: 开屏广告结束后的跳转

4. **getParamsFromNav(adType: AdType)**
   - 功能: 从导航栈中获取指定类型广告的请求参数

## 五、关键技术点

### 5.1 广告加载流程
```
1. 创建AdLoader对象
   ↓
2. 设置AdLoadListener监听器
   ↓
3. 构建AdRequestParams请求参数
   ↓
4. 调用loadAd发起请求
   ↓
5. 回调onAdLoadSuccess/onAdLoadFailure
   ↓
6. 展示广告（showAd）
```

### 5.2 状态监听机制
使用HarmonyOS的公共事件（CommonEvent）机制订阅广告状态变化：
- 激励广告: `com.huawei.hms.pps.action.PPS_REWARD_STATUS_CHANGED`
- 插屏广告: `com.huawei.hms.pps.action.PPS_INTERSTITIAL_STATUS_CHANGED`

### 5.3 广告类型区分
根据`AdType`枚举值，采用不同的处理策略：
- **激励/插屏广告**: 加载成功后立即展示
- **横幅/原生广告**: 加载后存入数组，在页面中通过组件展示
- **开屏广告**: 独立全屏页面展示，支持超时处理
